南平二儿子写给王正香的一封信

Posted on 2019-05-06 | Edited on 2019-05-09 

        

        

第一次写信
母亲您好，这是儿子平生第一次有给你写信的想法，早年间读书的时候，麻高（麻丘高级中学）初二上学期写过一次，严格来说那个顶多算由于考的不好，老师要求我们给来开会的家长写几句话，那一次的感受，你偷偷的跟着说过，有点受不了，至于为什么受不呢，我想多半是爱，这个词不常处于平民百姓家庭中，也不常于母子之间，在我看来，这样的家庭在中国不在少数。


为什么要写信
信本来想让嫂子读给您听，但是总怕表达不出儿子想对你说的话和情绪，思来想去，还是儿子亲自给您读， 为什么会想到给您读信呢？一来您在和病魔做抗争，需要孩子们的激励，按理说平时微信和你视频的时候，和你多说说话，也是可以激励你的，由于视频多了，您也有时候会比较劳累，儿子有时候也不知道和你聊些什么，总是那些家长里短，小朋友喝汤；二来呀，写信这类东西，比较容易保存，容易时常拿来看；正好前几天，嫂子问我爱奇艺会员的事，我说过几天母亲节诶，嫂子记得，说：父亲节，母亲节都给爸爸妈妈买了节日礼物，给你买个金花项链带；


和嫂子交流过，说给你写封信，顺便送朵花，当然了这里说的送朵花，不仅仅是送一笑朵花，主要是显得矫情，嫂子说：写信，她那看得懂呀，之前就想到了，只是没说全而已，我正好前几年看过腾讯视频的一档节目”见字如面”，节目的内容我在家也播放过，节目的内容是通过其朗读者读一些历史信和现代信，每期的主题还不一样，爱情呀，思念呀，家国情怀呀！儿子觉得写的这封信应该算家书，算儿子写给父母的那类。


母亲您对这个家庭是有多重要，您怎么可能不知道
受您影响，我们家的家风可唯非常的好，从来不投机打吧，从来不以大欺小，从来不人云亦云，不做傻事，你终于把两个儿子培养成人了，您吃的苦，我们是真知道，打那小，5 6岁左右，我在观田小学读学前班，那时还没有菜市场，我记得你摊位摆在观田小学右边路口。 那时的马路是真脏，有水有泥巴的， 


后面建了新菜场，你买了个摊位，好多了，你的摊位就在西北方向，那会儿下课了我常去，早上也去，为什么去您哪儿呢，当然去拿钱吃早餐，或者拿钱买吃的，那会儿我就感受到生活的不容易，从不多拿， 有时候又贪吃，多要了点，吃完又觉得没劲，还惹你唠叨，我觉得那会儿我就知道你的不易，家庭的不易，当然了，由于知识面的浅薄，哪知道这个地球上还分不同的国家，老外他们就过的没这么艰苦，砸门国家什么时候，老百姓能过上这样的日子。


再后来就是你出车祸的事啦，我记得是夏天，6点不到的样子，被三舅妈叫醒，稀里糊涂的，我记得那会儿，三舅妈，说你妈被撞了，你们要自己去上课，给你们送早餐钱来，我当时也不知道什么个情况，就是感觉不是很开心，大清早的，一顿数落，能感觉的言外之意是，你暂时见不到你妈啦！后面读了几天书学校的老师都在议论，那个后罗买菜的谁谁被个后八轮给撞了，再后来就是我拿着学校退给我的什么费用，24块多，还是19块7，有点忘了， 叫我去外婆家吃饭，说你妈去外婆家了，我记得那会儿，你躺在床上，我拿着钱给你，你还说了几句，具体说的啥记不清楚了，大概是说都知道了，不过那会儿，我觉得你的精神状态挺好的，再后来的我就不记得了，只隐隐约约的记得，你腿里有钢板，后面又取出了，我还记得那钢板的样子，长方形的，二十公分不到的样子，比较重，老开玩笑拿去卖。


网瘾少年
再后来呀，我的记忆就停留在网络游戏中，少年的大部分时间多半是在网吧度过的，搞笑的事，你有天觉得我在家那副郁郁寡欢的样，问我是不是想去上网，用南昌话是不是恋到了，我说没意思，你给我五块钱，我当时觉得上不过瘾，我打量着说，想通个宵，那样上的久点，你看着我，反正僵持了好久，你最后同意了，我记得那天通完宵回来睡觉，你在洗衣服，看着我回来，什么都没说，就说了一句，过了瘾了，


那时咱娘俩就了解的深，你总是能看穿儿子们的心思，就像长大后，我总能看穿你的小心思，我觉得哈，这个是咱娘俩你从小培养起来的，都不需要废话，使个眼神就知道你想干嘛，有啥想做的， 不过还是得夸你，就没见过像你这么通情达理的妈，在哥的心理估摸也是。


厉害极了
说了这么多，该夸你了，爸爸那德行是从小吃不得苦的，而您可厉害了，厉害极了，买菜，卖花片草纸，卖鱼，包括你小时候跑几十里地卖冰棒，拿到现在来，多识几个字，以你这胆量，又懂得人情世故，混个中国女首富，不是难事，我那会儿，就爱跟着你，或者去看你卖菜，一来看你生意咋样，生意好呢，你就开心，我也就开心，不然玩起来老惦记着；二来陪你说说话，坐久了，你就说:老子，且归，，再后面就是几点来收摊子，再后来呀，就是帮你送个饭，前几年的你不还是顶着37，8度的太阳，中午能打个顿，放在现在来看，那可是高温作业，辛苦的很， 现在好了，哪些生意你也不做了，继续培养下一代，妹妹，罗加陈，算我们的第三代了，我觉得的呀！还是你厉害，厉害极了，这两个小孩感觉比我们小时候聪明的多，这算是一代比一代强，记得有年，我们家做酒，三舅舅问我：”你聪聪、陈敏生了一男一女叫犀利波”，我呀！当然答不出来，后面呀，我才知道在你们长辈面前看到字辈生一男一女，那是好呀！说不出的好呀！


突围
妈，现在我跟你讲讲咱们国家的形式，整个大环境的形式，


中国经济这两年，也许还包括未来的几年，基本就处在一个突围的状态。为什么要突围呢，我们被什么力量围住了呢？


主要是两个力量，一个是全球格局正发生所谓百年未有之大变局，中国正当其中，被围住了，另一个力量是我们自己多年的高速增长带来的国内经济的一些变化，这也把我们围住了。那么，能不能完成这次突围，关系到中国能不能晋升到现代化强国的行列，同时也对全球经济走向会产生一些外延式的影响。


之前的中国为什么发展那么快，那是因为改革开发之前，我们国家就穷，改革开发之后，我们越穷，发展优势越大，我们劳动力便宜呀！很多老外都来中国搬厂，尤其是沿海地区，港口遍历周边；现在我们中国发展好了，已经是 GDP 第二大国，但是呀，优势也没了；那你们的行话叫工价贵的很，不如印度，越南这些相对发展慢的国家，他们劳动力便宜呀！加上美国和中国近些年天天搞贸易战，就是想遏制中国的发展，现在我们的国家处于不上不下的地步，所以呀，现在国家老是提改革，说道改革呀，说白了，就是高科技产业和其他相关产业要有创新，这些创新又需要新型教育人才去做这些事，加上近些年我们中国出生率于越来越低了，出生率已经出现负增长的局面，这说明呀，中国的未来几十年老龄化加剧，所以更需要各方面的创新，才能破解这个僵局，未来深圳-后海地区应该会是作为人民币外汇进入国际市场的主要地区，深圳的发展，也会继续加快，儿子学历在大城市算低的，但是呀，儿子是您培养出来的，儿子我勤劳呀，不笨呀，愿意跟着国家的城市化发展区一起加油，一起突围呀！
母亲呀，你也需要突围呀，去战胜这个小小的 “鼻癌”，继续帮我们的国家去培养突围的新型人才，继续帮我们罗家去培养勤劳，勇敢，不屈不挠的后代，中国突围需要咱娘俩！

前言
鄙人南下就业，以为掌握 Dubbo、SpringCloud 等技术栈就比较占优，不曾想到各种碰壁，事与愿违，在准备面试题的过程中，开始思考，痛并思痛，自己是不是该改变学习策略了；框架和基础知识那个更重要呢？


第一家面试的是亚马逊中国，上来就是一顿算法题，懵了；


第二家面试的是懒人科技，上来也是一顿 Java 基础知识，GC 优化，多线程、并发编程，底层原理，哎… 又被打败了；


刚好看到 极客时间 有相关的文章，甚是欣喜，特意引用，以史为诫。


极客时间 | 技术人员应该停止学习框架，专注基础知识
作为一名开发人员，需要不停地学习新技术。那么，我们应该学习什么样的技术？关于这个话题，有着 15 年工作经验的软件工程师爱德华·兹西佐夫（Eduards Sizovs）发文表达了自己的看法。


他认为，开发者需要掌握最新的技术，每天学习编程语言、框架和库，知道的工具越新越好，但这一切都是在浪费时间。科技以光速变化，这场比赛没有赢家，因为它没有终点。


技术的未来预期寿命与其当前存在时间成正比，每一段额外的生存期都意味着更长的剩余预期寿命。技术存在的时间越长，学习它就越安全，对于技术人员来说，不要急于学习新技术，因为它有很高的消亡概率；时间是最好的导师，它会证明哪些技术值得学习，所以需要学会等待。


曾经，兹西佐夫的导师给他上了重要一课。他的导师询问他正在做什么，他回答正在阅读一本关于如何用 GWT 构建现代 Java 应用程序的书，GWT 是必须跟进的技术。在此之前，他还读了一本 500 页的关于 Apache Tapestry 的书，但他现在根本没用到 Apache Tapestry ，这也没办法帮助他很好地解决 GWT 相关的问题。


因此，导师建议他正确地设置优先级，把 80% 的时间花在基础学习上，然后剩下 20% 的时间留给框架，库和工具的学习即可。


现如今，十年过去了，他经历了 50 个不同的软件项目。上述这些建议让他学到的所有东西都可以跨公司、团队、跨领域使用。他认为，只有深入研究项目本质，才会发现它们都是相似的：


    

  1. 编程语言是不同的，但设计是相似的。
    2. 

  2. 框架是不同的，但设计模式是可以通用的。
    3. 

  3. 开发者是不同的，但与人打交道的规则是统一的。
    4. 



框架、库和工具是会变化的，但时间是宝贵的。开发者应该将宝贵的时间花在可移植的技能上，比如框架进化体系结构，干净的代码，设计模式，DDD、安全精简编码原则、容错模式、HTTP 和 REST 等技能上。

Tomcat 三种线程模式


一、bio(blocking I/O)


即阻塞式I/O操作，表示Tomcat使用的是传统的Java I/O操作(即java.io包及其子包)。是基于JAVA的HTTP/1.1连接器，Tomcat7以下版本在默认情况下是以bio模式运行的。一般而言，bio模式是三种运行模式中性能最低的一种。我们可以通过Tomcat Manager来查看服务器的当前状态。（Tomcat7或以下，在 Linux 系统中默认使用这种方式）


一个线程处理一个请求，缺点：并发量高时，线程数较多，浪费资源


二、nio(new I/O)


是Java SE 1.4及后续版本提供的一种新的I/O操作方式(即java.nio包及其子包)。Java nio是一个基于缓冲区、并能提供非阻塞I/O操作的Java API，因此nio也被看成是non-blocking I/O的缩写。它拥有比传统I/O操作(bio)更好的并发运行性能。要让Tomcat以nio模式来运行只需要在Tomcat安装目录/conf/server.xml 中将对应的中protocol的属性值改为 org.apache.coyote.http11.Http11NioProtocol即可


利用 Java 的异步请求 IO 处理，可以通过少量的线程处理大量的请求


注意： Tomcat8 以上版本在 Linux 系统中，默认使用的就是NIO模式，不需要额外修改 ，Tomcat7必须修改Connector配置来启动


undefined


三、apr(Apache Portable Runtime/Apache可移植运行时) （ 安装配置过程相对复杂）


Tomcat将以JNI的形式调用Apache HTTP服务器的核心动态链接库来处理文件读取或网络传输操作，从而大大地提高Tomcat对静态文件的处理性能。Tomcat apr也是在Tomcat上运行高并发应用的首选模式。从操作系统级别来解决异步的IO问题

回答
如果Action不符合CRUD操作那该怎么办？


    

  1. 重新构造这个Action，使得它像一个资源的field（我理解为部分域或者部分字段）。这种方法在Action不包含参数的情况下可以奏效。例如一个有效的action可以映射成布尔类型field，并且可以通过PATCH更新资源。
    2. 

  2. 利用RESTful原则像处理子资源一样处理它。例如，Github的API让你通过PUT /gists/:id/star 来 star a gist ，而通过DELETE /gists/:id/star来进行 unstar 。
    3. 

  3. 有时候你实在是没有办法将Action映射到任何有意义的RESTful结构。例如，多资源搜索没办法真正地映射到任何一个资源接入点。这种情况，/search 将非常有意义，虽然它不是一个名词。这样做没有问题 - 你只需要从API消费者的角度做正确的事，并确保所做的一切都用文档清晰记录下来了以避免（API消费者的）困惑。

CIA 原则
所有互联网企业都要建立一套完整的信息安全体系，遵循 CIA 原则，即保密性（Confidentiality）、完整性（Integrity）、可用性（Availability）


    

  • 保密性。对需要保护的数据（比如用户的私人信息等）进行保密操作，无论是存储还是传输，都要保证用户数据及相关资源的安全。比如，在存储文件时会进行加密，在数据传输中也会通过各种编码方式进行加密等。在实际编程中，通常使用加密等手段保证数据的安全。黑客不只是外部的，有可能从内部窃取数据，所以现在大多数企业的用户名敏感信息都不是以明文存储的，避免数据管理员在某些利益的驱动下，直接托库下载。数据泄露可能导致黑客进一步利用这些数据进行网站攻击，造成企业的巨大损失。
